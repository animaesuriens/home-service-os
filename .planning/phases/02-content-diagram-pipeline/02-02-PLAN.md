Create `linkedin-posts/templates/` directory, then create `linkedin-posts/templates/diagram.html`.

This is a self-contained HTML file that renders a flowchart diagram. It receives data via template placeholders that the renderer replaces before passing to Playwright. Per D-16, this is ONE reusable template for all platforms and layouts.

**Template placeholders** (replaced by diagram-renderer.js before rendering):
- `{{PRIMARY_COLOR}}` - Platform hex color (e.g., `#7B2D8E`)
- `{{PLATFORM_NAME}}` - Platform display name (e.g., `Make`)
- `{{TITLE}}` - Bundle title (e.g., `Lead Capture & Qualification`)
- `{{STEPS_JSON}}` - JSON array of step objects `[{label, actor}]`
- `{{LAYOUT}}` - Either `vertical` or `swimlane`

**CSS Design Requirements (per D-15, D-17, D-18):**
- Body: exactly 1080px wide x 1350px tall, box-sizing: border-box
- Background: subtle gradient using platform color at low opacity (e.g., `linear-gradient(180deg, {primary}08 0%, {primary}15 100%)`)
- Platform brand bar at top: 60px height, full-width, solid `--primary-color` background, white text showing platform name
- Title area below brand bar: large bold text, centered, showing the bundle title
- Steps area: centered column of step nodes with connectors between them
- Step nodes: white background, 3px solid `--primary-color` border, rounded corners (12px), drop shadow, padding 16px 24px, font-size 16px, font-weight 600, max-width 700px, centered text
- Connectors between steps: 3px wide x 24px tall colored bars in `--primary-color`
- Arrow indicator on connectors: small downward triangle/chevron at bottom of connector
- Font: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- Step numbering: each step shows its number (1, 2, 3...) in a small circle on the left side of the step node, filled with `--primary-color`, white text

**Swimlane Layout (per D-19):**
When `layout === 'swimlane'`, group steps by their `actor` field into vertical lanes side by side:
- Lane headers: uppercase text in `--primary-color`, font-size 13px, letter-spacing 1.5px, font-weight 700
- Lanes arranged horizontally using CSS flexbox, each lane min-width 280px
- Steps within each lane flow vertically with connectors
- Horizontal dotted connector lines between lanes at transfer points (steps that hand off to another actor)
- Maximum 3 lanes to fit in 1080px width

**Vertical Layout:**
When `layout === 'vertical'`, all steps in a single centered column top-to-bottom with connectors.

**Footer area:** Small text at bottom `"Powered by {{PLATFORM_NAME}}"` in `--primary-color`

**JavaScript in the template:**
The template includes a `<script>` block that:
1. Parses `{{STEPS_JSON}}` (already a JS literal after replacement)
2. Reads `{{LAYOUT}}` to determine which rendering mode
3. For vertical: creates step nodes with numbered circles and connectors between them
4. For swimlane: groups steps by `actor`, creates lane divs with headers, renders steps within each lane
5. All DOM creation is vanilla JS (no frameworks needed)

The complete HTML structure:
```
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <style>
    :root { --primary-color: {{PRIMARY_COLOR}}; }
    /* All styles inline in the template */
  </style>
</head>
<body>
  <div class="diagram-container">
    <div class="brand-bar">{{PLATFORM_NAME}}</div>
    <div class="title">{{TITLE}}</div>
    <div class="flowchart" id="flowchart"></div>
    <div class="footer">Powered by {{PLATFORM_NAME}}</div>
  </div>
  <script>
    const steps = {{STEPS_JSON}};
    const layout = '{{LAYOUT}}';
    // DOM generation code
  </script>
</body>
</html>
```

**CRITICAL:** All CSS must be inline in the `<style>` tag (no external stylesheets). The template must be completely self-contained since Playwright loads it via `setContent()` with no network access.

Create `linkedin-posts/lib/diagram-renderer.js` as a CommonJS module (per existing project convention).

**Exports:**
- `renderDiagram(config, outputPath)` - Renders a single diagram to PNG
- `renderAllDiagrams(bundles, outputDir)` - Renders diagrams for all bundles across all 3 platforms
- `PLATFORM_COLORS` - Color constants

**PLATFORM_COLORS constant per D-17:**
```javascript
const PLATFORM_COLORS = {
  make:   { primary: '#7B2D8E', name: 'Make' },
  zapier: { primary: '#FF4A00', name: 'Zapier' },
  n8n:    { primary: '#9FD7A0', name: 'n8n' }
};
```

**renderDiagram(config, outputPath) function:**

Parameters:
- `config`: `{ platform: 'make'|'zapier'|'n8n', title: string, steps: [{label, actor}], layout: 'vertical'|'swimlane' }`
- `outputPath`: absolute or relative path for output PNG file

Implementation:
1. Read `templates/diagram.html` from `path.join(__dirname, '..', 'templates', 'diagram.html')`
2. Replace placeholders in the HTML string:
   - `{{PRIMARY_COLOR}}` -> `PLATFORM_COLORS[config.platform].primary`
   - `{{PLATFORM_NAME}}` -> `PLATFORM_COLORS[config.platform].name`
   - `{{TITLE}}` -> `config.title`
   - `{{STEPS_JSON}}` -> `JSON.stringify(config.steps)`
   - `{{LAYOUT}}` -> `config.layout`
3. Launch Chromium headless via `require('playwright').chromium.launch({ headless: true })`
4. Create new page with viewport `{ width: 1080, height: 1350 }` and `deviceScaleFactor: 2` per D-18
5. Set page content with `await page.setContent(html, { waitUntil: 'networkidle' })`
6. Wait 500ms for fonts/rendering to settle: `await page.waitForTimeout(500)`
7. Ensure output directory exists: `await fs.ensureDir(path.dirname(outputPath))`
8. Screenshot: `await page.screenshot({ path: outputPath, type: 'png', fullPage: false })`
9. Close page (NOT browser -- browser lifecycle managed by caller or renderAllDiagrams)
10. Return `{ outputPath, platform: config.platform, width: 1080, height: 1350 }`

**renderAllDiagrams(bundles, outputDir) function:**

Parameters:
- `bundles`: array of bundle objects from bundles.json
- `outputDir`: base output directory (e.g., `linkedin-posts/posts`)

Implementation:
1. Launch browser ONCE: `const browser = await chromium.launch({ headless: true })`
2. Read template ONCE into memory
3. For each bundle, for each platform ('make', 'zapier', 'n8n'):
   - Build output path: `path.join(outputDir, bundle.journeySlug, `${platform}.png`)`
   - Create page with viewport settings
   - Replace placeholders and render
   - Take screenshot
   - Close page
   - Log: `console.log('Generated: ' + outputPath)`
4. Close browser after ALL diagrams are done
5. Return array of `{ bundleId, platform, outputPath }` for all generated diagrams

**Error handling:**
- Wrap each diagram render in try/catch
- On error, log `console.error('Failed to render ${bundle.id}/${platform}: ${err.message}')` and continue to next
- Return both successes and failures in the result

**Critical implementation notes:**
- Use `require('playwright')` not `import` (CommonJS project)
- Template file path uses `path.join(__dirname, '..', 'templates', 'diagram.html')` for reliable resolution
- Browser is launched ONCE and reused for all renders (much faster than per-diagram launch)
- Each diagram gets its OWN page (prevents state leakage between renders)
- Per D-18: actual PNG pixel dimensions will be 2160x2700 (1080x1350 * deviceScaleFactor 2)
