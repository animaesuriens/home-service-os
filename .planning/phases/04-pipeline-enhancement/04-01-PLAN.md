<objective>
Add --yaml flag to pipeline scripts for single-file YAML processing.

Purpose: Enable per-YAML processing mode so Phases 5-12 can process one YAML at a time instead of all 8, reducing context size and preventing hallucination. This implements the core architectural change for v1.1.
Output: All four pipeline scripts (parse-yaml, analyze-processes, generate-bundles) accept --yaml <filename> to filter processing to a single YAML file. Backward compatible -- no flag means all files.
</objective>
